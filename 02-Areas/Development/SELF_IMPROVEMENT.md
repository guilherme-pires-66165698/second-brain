# Second Brain Self-Improvement Guide

## How This Knowledge Base Grows

### 1. End of Session Capture
After each coding session, use the prompt in `00-Inbox/CAPTURE_PROMPT.md` to extract learnings.

### 2. Weekly Review
- Process items in 00-Inbox/
- Move to correct folders
- Update existing docs with new insights
- Remove outdated information

### 3. What to Capture

| Type | Where to Save |
|------|---------------|
| New code pattern | 03-Resources/Languages/[lang].md |
| Framework trick | 03-Resources/Frameworks/[framework].md |
| Project decision | 01-Projects/[project].md |
| Bug pattern | 02-Areas/Development/common-bugs.md |
| Tool config | 03-Resources/Tools/[tool].md |
| Coding standard | 02-Areas/Development/coding-standards.md |

### 4. Prompts for AI Assistants

**Session End:**
"Capture learnings from this session to my second-brain repo"

**Before Coding:**
"Read my second-brain README for my coding preferences"

**Code Review:**
"Check my coding-standards.md and review this code"

### 5. Keep It Useful
- Delete outdated info
- Consolidate duplicates
- Keep examples minimal but complete
- Update when better patterns emerge
